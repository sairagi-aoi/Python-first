# 部署番号チェックツール 設計メモ

## 目的

入力された部署コードが登録済みか確認する。
未登録の場合は、上司判断で修正・差し戻し・中止を行う。
最終結果はCSVログに保存する。

## 入力

- 部署コード
- 上司判断の選択番号
- 修正後の部署コード

## 出力

### 画面出力

- 起動時のタイトルと説明
- 部署コード入力の案内
- OK / NG の判定結果
- 類似候補
- 上司判断メニュー
- 登録済み部署一覧
- 差し戻し案内
- 中止メッセージ
- 修正完了メッセージ

### ファイル出力

- `dept_check_log.csv` に処理結果を1行保存する

## 保存するログ項目

- `timestamp`: 処理日時
- `input_raw`: 最初に入力された値
- `input_norm`: 正規化後の入力値
- `initial_result`: 初回判定結果
- `choice`: 上司判断の選択
- `corrected_raw`: 修正入力された値
- `corrected_norm`: 修正入力を正規化した値
- `final_result`: 最終結果
- `note`: 補足メモ

## 処理の流れ

1. 部署番号チェックツールの見出しとメインの案内を表示する
2. ユーザーに部署番号の入力を求める
3. 現在日時をtsに代入する
4. CSVの共通ログの土台を作成する
5. 入力された内容が部署マスタに存在すれば

## 必要な関数

- `now_str()`
  - 現在日時を文字列で返す

- `append_log(row)`
  - CSVにログを1行追加する

- `normalize_dept_id(raw)`
  - 入力値の前後空白を削除し、小文字にする

- `show_help()`
  - 差し戻し時の確認手順を表示する

- `show_valid_depts()`
  - 登録済み部署一覧を表示する

- `suggest_similar(dept_id)`
  - 似ている部署コードを表示する

- `prompt_dept_id()`
  - 部署コードを入力してもらい、rawと正規化済みIDを返す

- `supervisor_decision()`
  - 上司判断の選択肢を受け付ける

- `main()`
  - 全体の処理を進める

## テスト項目

- `jimuijika` を入力するとOKになる
- `KANGO1NS` を入力するとOKになる
- `  kango2ns  ` を入力するとOKになる
- 未登録コードを入力するとNGになる
- 類似候補が表示される
- `0` を選ぶと中止できる
- `1` を選ぶと修正入力できる
- `2` を選ぶと差し戻しできる
- `3` を選ぶと部署一覧が表示される
- CSVログが保存される
